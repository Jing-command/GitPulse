# GitPulse 数据库设计文档

## 一、数据库概述

### 1.1 数据库选型

| 项目 | 选型 | 说明 |
|------|------|------|
| 主数据库 | PostgreSQL 15+ | 功能强大，支持 JSON 类型 |
| 备选数据库 | MySQL 8.0+ | 兼容支持 |
| ORM | Prisma 5.x | 类型安全，迁移管理方便 |

### 1.2 命名规范

遵循腾讯代码规范和 RESTful API 设计规范：

| 规范项 | 说明 | 示例 |
|--------|------|------|
| 表名 | 小写、下划线分隔、复数形式 | `projects`、`content_versions` |
| 字段名 | 小写、下划线分隔 | `created_at`、`project_id` |
| 索引名 | `idx_表名_字段名` | `idx_projects_name` |
| 外键名 | `fk_表名_关联表名` | `fk_contents_projects` |
| 主键 | 使用 `cuid()` 生成 | `clx1234567890` |

## 三、数据表详细设计

### 3.1 用户表 (users)

存储系统用户信息。

```prisma
model User {
  // 主键，使用 cuid 生成
  id        String   @id @default(cuid())
  
  // 用户邮箱，唯一
  email     String   @unique
  
  // 用户名称
  name      String?
  
  // 密码（bcrypt 加密存储）
  password  String?
  
  // 全局角色：admin / editor / viewer
  role      String   @default("viewer")
  
  // OAuth 提供商
  oauth_provider String?
  
  // OAuth 用户 ID
  oauth_id       String?
  
  // 创建时间
  created_at DateTime @default(now())
  
  // 更新时间
  updated_at DateTime @updatedAt
  
  // 关联关系
  contents      ContentVersion[]
  approvals     Approval[]
  project_members ProjectMember[]
  
  // 索引
  @@index([email])
  @@index([oauth_provider, oauth_id])
  @@map("users")
}
```

**字段说明**：

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| id | string | 是 | 主键，cuid 格式 |
| email | string | 是 | 用户邮箱，唯一 |
| name | string | 否 | 用户名称 |
| password | string | 否 | bcrypt 加密后的密码 |
| role | string | 是 | 全局角色，默认 viewer |
| oauth_provider | string | 否 | OAuth 提供商（github/gitlab） |
| oauth_id | string | 否 | OAuth 用户 ID |
| created_at | datetime | 是 | 创建时间 |
| updated_at | datetime | 是 | 更新时间 |

**索引设计**：

| 索引名 | 字段 | 类型 | 说明 |
|--------|------|------|------|
| PRIMARY | id | 主键 | 主键索引 |
| users_email_key | email | 唯一 | 邮箱唯一索引 |
| users_oauth_idx | oauth_provider, oauth_id | 普通 | OAuth 登录查询 |

### 3.2 项目表 (projects)

存储项目信息。

```prisma
model Project {
  // 主键
  id          String   @id @default(cuid())
  
  // 项目名称
  name        String
  
  // 项目描述
  description String?
  
  // 仓库地址
  repo_url    String?
  
  // 项目配置（JSON 格式）
  config      Json?
  
  // 创建时间
  created_at  DateTime @default(now())
  
  // 更新时间
  updated_at  DateTime @updatedAt
  
  // 关联关系
  commits     Commit[]
  contents    Content[]
  contexts    Context[]
  members     ProjectMember[]
  
  // 索引
  @@index([name])
  @@index([repo_url])
  @@map("projects")
}
```

**config 字段结构**：

```json
{

  "content": {
    "languages": ["zh", "en"],
    "formats": ["markdown", "json", "html", "pdf"]
  },
  "docs": {
    "framework": "vitepress",
    "output": "./docs",
    "sidebar": "./docs/.vitepress/sidebar.ts"
  },
  "review": {
    "enabled": true,
    "auto_publish": false
  }
}
```

### 3.3 项目成员关联表 (project_members)

存储项目与用户的关联关系。

```prisma
model ProjectMember {
  // 项目 ID
  project_id String
  
  // 用户 ID
  user_id    String
  
  // 项目角色：admin / editor / viewer
  role       String   @default("viewer")
  
  // 加入时间
  joined_at  DateTime @default(now())
  
  // 关联关系
  project    Project  @relation(fields: [project_id], references: [id], onDelete: Cascade)
  user       User     @relation(fields: [user_id], references: [id], onDelete: Cascade)
  
  // 复合主键
  @@id([project_id, user_id])
  @@map("project_members")
}
```

### 3.4 Commit 分析表 (commits)

存储 Git commit 分析结果。

```prisma
model Commit {
  // 主键
  id            String   @id @default(cuid())
  
  // Commit SHA
  hash          String   @unique
  
  // Commit 消息
  message       String
  
  // 作者名称
  author        String
  
  // 作者邮箱
  author_email  String?
  
  // 提交时间
  timestamp     DateTime
  
  // 影响级别：major / minor / patch
  impact_level  String
  
  // 变更摘要（JSON 格式）
  summary       Json
  
  // 项目 ID
  project_id    String
  
  // 创建时间
  created_at    DateTime @default(now())
  
  // 关联关系
  project       Project  @relation(fields: [project_id], references: [id], onDelete: Cascade)
  contents      Content[]
  
  // 索引
  @@index([hash])
  @@index([project_id])
  @@index([timestamp])
  @@map("commits")
}
```

**summary 字段结构**：

```json
{
  "type": "feature",
  "scope": ["content-engine", "ai"],
  "keywords": ["AI", "文案生成", "GPT"],
  "breaking": false,
  "changes_count": 5,
  "files_changed": 12,
  "additions": 350,
  "deletions": 50
}
```

### 3.5 内容表 (contents)

存储生成的内容。

```prisma
model Content {
  // 主键
  id          String   @id @default(cuid())
  
  // 内容类型：changelog / technical / seo
  type        String
  
  // 标题
  title       String
  
  // 内容（Markdown 格式）
  content     String   @db.Text
  
  // 多格式输出（JSON 格式）
  formats     Json?
  
  // 元数据（JSON 格式）
  metadata    Json?
  
  // 语言
  language    String   @default("zh")
  
  // 状态：draft / pending_review / approved / published / archived
  status      String   @default("draft")
  
  // 关联的 commit ID
  commit_id   String?
  
  // 项目 ID
  project_id  String
  
  // 创建时间
  created_at  DateTime @default(now())
  
  // 更新时间
  updated_at  DateTime @updatedAt
  
  // 发布时间
  published_at DateTime?
  
  // 关联关系
  project     Project   @relation(fields: [project_id], references: [id], onDelete: Cascade)
  commit      Commit?   @relation(fields: [commit_id], references: [id])
  versions    ContentVersion[]
  approvals   Approval[]
  
  // 索引
  @@index([project_id])
  @@index([type])
  @@index([status])
  @@index([language])
  @@index([created_at])
  @@map("contents")
}
```

**formats 字段结构**：

```json
{
  "markdown": "# v1.2.0 更新日志\n...",
  "json": "{\"title\": \"v1.2.0 更新日志\", ...}",

}
```

**metadata 字段结构**：

```json
{
  "keywords": ["新功能", "优化"],
  "summary": "本次更新带来了多项新功能...",
  "tags": ["v1.2.0", "feature"],
  "reading_time": 3,

### 3.6 内容版本表 (content_versions)

model ContentVersion {

  id          String   @id @default(cuid())

  content     String   @db.Text

  author_id   String
  
  // 创建时间
  created_at  DateTime @default(now())

  author      User     @relation(fields: [author_id], references: [id])
  
  // 唯一约束
  @@unique([content_id, version])
  @@index([content_id])
  @@map("content_versions")
}
```

## 四、完整 Prisma Schema

```prisma
// prisma/schema.prisma

generator client {
  provider = "prisma-client-js"
}

  provider = "postgresql"

  id             String   @id @default(cuid())
  email          String   @unique
  name           String?
  password       String?
  role           String   @default("viewer")
  oauth_provider String?
  oauth_id       String?
  created_at     DateTime @default(now())
  updated_at     DateTime @updatedAt
  
  contents       ContentVersion[]
  approvals      Approval[]
  project_members ProjectMember[]
  
  @@index([email])
  @@index([oauth_provider, oauth_id])
  @@map("users")
}

model Project {
  id          String   @id @default(cuid())
  name        String
  description String?
  repo_url    String?
  config      Json?
  created_at  DateTime @default(now())
  updated_at  DateTime @updatedAt
  
  commits     Commit[]
  contents    Content[]
  contexts    Context[]
  members     ProjectMember[]
  
  @@index([name])
  @@index([repo_url])
  @@map("projects")
}

model ProjectMember {
  project_id String
  user_id    String
  role       String   @default("viewer")
  joined_at  DateTime @default(now())
  
  project    Project  @relation(fields: [project_id], references: [id], onDelete: Cascade)
  user       User     @relation(fields: [user_id], references: [id], onDelete: Cascade)
  
  @@id([project_id, user_id])
  @@map("project_members")
}

model Commit {
  id           String   @id @default(cuid())
  hash         String   @unique
  message      String
  author       String
  author_email String?
  timestamp    DateTime
  impact_level String
  summary      Json
  project_id   String
  created_at   DateTime @default(now())
  
  project      Project  @relation(fields: [project_id], references: [id], onDelete: Cascade)
  contents     Content[]
  
  @@index([hash])
  @@index([project_id])
  @@index([timestamp])
  @@map("commits")
}

model Content {
  id           String   @id @default(cuid())
  type         String
  title        String
  content      String   @db.Text
  formats      Json?
  metadata     Json?
  language     String   @default("zh")
  status       String   @default("draft")
  commit_id    String?
  project_id   String
  created_at   DateTime @default(now())
  updated_at   DateTime @updatedAt
  published_at DateTime?
  
  project      Project   @relation(fields: [project_id], references: [id], onDelete: Cascade)
  commit       Commit?   @relation(fields: [commit_id], references: [id])
  versions     ContentVersion[]
  approvals    Approval[]
  
  @@index([project_id])
  @@index([type])
  @@index([status])
  @@index([language])
  @@index([created_at])
  @@map("contents")

  author_id  String

  reviewer    User     @relation(fields: [reviewer_id], references: [id])

  @@index([reviewer_id])

## 五、索引策略

### 5.1 主键索引

所有表使用 `cuid()` 作为主键，自动创建主键索引。

### 5.2 唯一索引

| 表 | 字段 | 说明 |
|----|------|------|
| users | email | 邮箱唯一 |
| commits | hash | Commit SHA 唯一 |
| content_versions | content_id + version | 版本号唯一 |

### 5.3 普通索引

| 表 | 字段 | 查询场景 |
|----|------|----------|
| users | oauth_provider, oauth_id | OAuth 登录 |
| projects | name | 项目名称搜索 |
| projects | repo_url | 仓库地址查询 |
| commits | project_id | 按项目查询 commits |
| commits | timestamp | 按时间排序 |
| contents | project_id | 按项目查询内容 |
| contents | type | 按类型筛选 |
| contents | status | 按状态筛选 |
| contents | language | 按语言筛选 |
| contents | created_at | 按时间排序 |

### 6.2 种子数据

// prisma/seed.ts
import { PrismaClient } from '@prisma/client';

const prisma = new PrismaClient();

async function main() {
  // 创建默认管理员
  await prisma.user.upsert({
    where: { email: 'admin@gitpulse.dev' },
    update: {},
    create: {
      email: 'admin@gitpulse.dev',
      name: 'Admin',
      password: await hash('Admin@123', 10),
      role: 'admin',
    },
  });
}

main()
  .catch((e) => {
    console.error(e);
    process.exit(1);
  })
  .finally(async () => {
    await prisma.$disconnect();
  });
```

## 七、性能优化建议

### 7.1 查询优化

- 使用 `select` 只查询需要的字段
- 使用 `include` 预加载关联数据
- 使用 `take` 和 `skip` 实现分页
- 避免 N+1 查询问题

### 7.2 写入优化

- 使用事务处理批量操作
- 使用 `createMany` 批量插入
- 合理使用 `upsert` 避免重复

### 7.3 连接池配置

```env
DATABASE_URL="postgresql://user:password@localhost:5432/gitpulse?connection_limit=10&pool_timeout=30"
```
